# 🏥 Hospital Data Analysis Dashboard (MySQL + Power BI)

## 📌 Project Overview

This project analyzes hospital operational data using **MySQL for data processing** and **Power BI for visualization**.
The goal of the project is to explore patient activity, doctor performance, medical procedures, and revenue generation using simple SQL queries and interactive dashboards.

The dataset consists of **5 relational tables**, and SQL **views were created to prepare aggregated datasets** that were later used to build Power BI dashboards.

This project demonstrates practical skills in:

* SQL querying
* Relational database analysis
* Data modeling
* Business intelligence visualization

---

# 🗂 Dataset Structure

The dataset contains the following tables:

### 1. Patient

Stores patient details.

| Column    | Description               |
| --------- | ------------------------- |
| PatientID | Unique patient identifier |
| firstname | Patient first name        |
| lastname  | Patient last name         |
| email     | Patient email             |

---

### 2. Doctor

Stores doctor information.

| Column         | Description                     |
| -------------- | ------------------------------- |
| DoctorID       | Unique doctor identifier        |
| DoctorName     | Name of doctor                  |
| Specialization | Doctor's medical specialization |
| DoctorContact  | Contact information             |

---

### 3. Appointment

Stores appointment records between patients and doctors.

| Column        | Description           |
| ------------- | --------------------- |
| AppointmentID | Unique appointment ID |
| Date          | Appointment date      |
| PatientID     | Linked patient        |
| DoctorID      | Linked doctor         |

---

### 4. Medical_Procedure

Stores procedures performed during appointments.

| Column        | Description               |
| ------------- | ------------------------- |
| ProcedureID   | Unique procedure ID       |
| ProcedureName | Name of medical procedure |
| AppointmentID | Linked appointment        |

---

### 5. Billing

Stores billing information for patients.

| Column    | Description                  |
| --------- | ---------------------------- |
| InvoiceID | Billing invoice ID           |
| PatientID | Patient responsible for bill |
| Items     | Medical service billed       |
| Amount    | Billing amount               |

---

# 🏗 Project Architecture

The project follows a **3-layer architecture**.

### 1️⃣ Data Layer

Raw relational tables stored in **MySQL**.

Tables:

* Patient
* Doctor
* Appointment
* Medical_Procedure
* Billing

---

### 2️⃣ Business Logic Layer (SQL Views)

SQL views were created to simplify data aggregation before loading data into Power BI.

Views created:

* `vw_doctor_appointments`
* `vw_doctor_performance`
* `vw_doctor_revenue`
* `vw_patient_summary`
* `vw_hospital_dashboard`

These views handle joins and aggregations such as:

* Total appointments per doctor
* Total procedures performed
* Revenue per doctor
* Patient appointment activity
* Overall hospital KPIs

---

### 3️⃣ Visualization Layer

Power BI dashboards were built using the SQL views.

Power BI is used for:

* Interactive charts
* Filtering
* Data exploration
* Business insights
